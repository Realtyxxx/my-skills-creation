# NVCC Blackwell IR Extractor

A specialized utility for compiling CUDA code to the NVIDIA Blackwell (`sm_100` / `sm_100a`) architecture and extracting all Intermediate Representation (IR) artifacts.

## Purpose

Designed for low-level GPU kernel developers who need to verify how CUDA C++ code lowers into PTX and SASS on Blackwell. The script automatically applies the correct flags to preserve every compilation stage and runs `cuobjdump` to extract human-readable SASS.

## Features

- **Strict Architecture Focus**: Defaults to `sm_100` (overridable to `sm_100a`).
- **Full IR Pipeline**: Automatically applies `--save-temps`, `-lineinfo`, and `-Xptxas=-v`.
- **SASS Extraction**: Extracts `.sass` from all `.cubin` files using `cuobjdump`.
- **Clean Output**: All intermediate files (including `.o`) go to a dedicated output directory (`--output-dir`).
- **Prerequisite Check**: Validates `nvcc` and `cuobjdump` availability before running.

## Usage

```bash
python scripts/compile_and_dump_ir.py --input my_kernel.cu
```

### Options

| Flag                  | Default                 | Description                                 |
| --------------------- | ----------------------- | ------------------------------------------- |
| `--input` / `-i`      | _(required)_            | CUDA source file to compile                 |
| `--arch` / `-a`       | `sm_100`                | Target architecture (`sm_100` or `sm_100a`) |
| `--output-dir` / `-d` | `./blackwell_ir_output` | Directory for intermediate files            |
| `--options`           | _(none)_                | Additional `nvcc` flags to append           |

### Example

```bash
# Compile with architecture-specific features, custom output dir
python scripts/compile_and_dump_ir.py -i kernel.cu -a sm_100a -d ./ir_out
```

## What the Script Does

1. Checks that `nvcc` and `cuobjdump` are available in PATH.
2. Creates the output directory if it doesn't exist.
3. Runs `nvcc` with Blackwell-specific flags, redirecting intermediates to the output directory.
4. Extracts SASS from all generated `.cubin` files via `cuobjdump -sass`.
5. Lists all artifacts generated by **this run** (stale files from previous runs are excluded).
